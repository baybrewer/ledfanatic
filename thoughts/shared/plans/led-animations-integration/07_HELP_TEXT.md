# Phase 7 — Help Text and Instructions

## Goal

Every tab/page gets a collapsible instruction panel that explains what the page does and how to use it. First-time users should be able to understand the entire interface without external documentation.

## Design

### Help panel component

A reusable collapsible panel at the top of each tab:

```html
<div class="help-panel collapsed">
  <button class="help-toggle" aria-expanded="false">
    <span class="help-icon">?</span> How to use this page
  </button>
  <div class="help-content" hidden>
    <p>Instructions here...</p>
  </div>
</div>
```

### Behavior
- Collapsed by default (experienced users aren't slowed down)
- Click to expand/collapse
- `aria-expanded` for accessibility
- Remember collapse state in `localStorage` per tab
- Subtle styling — doesn't compete with controls

### Help text per tab

#### Live tab
```
This is your live control dashboard.

• The current scene name shows what's playing on the LEDs right now
• Presets are saved combinations of effect + parameters — tap to recall them
• Use the brightness slider above to control overall LED brightness
• "Blackout" turns all LEDs off immediately; "Resume" restores output
• The auto-brightness checkbox enables sunrise/sunset automation (requires location config)
```

#### Effects tab
```
Choose an animation for your LED pillar.

• Tap any effect to activate it immediately on the live LEDs
• Effects are grouped: Classic, Ambient, Sound-Reactive, and Built-in
• Sound-Reactive effects respond to music via the microphone (configure in Audio tab)
• When an effect is active, controls appear below:
  – Speed slider adjusts animation speed
  – Palette dropdown changes the color scheme
  – Other parameters vary per effect
• Animation Switcher: select multiple effects and the pillar will automatically
  cycle between them with smooth cross-fade transitions
```

#### Media tab
```
Upload and play images, GIFs, and videos on the pillar.

• Tap "Upload Media" to add files from your phone or computer
• Supported formats: PNG, JPG, GIF, MP4, WebM
• Uploaded media is transcoded to fit the 10×172 LED grid
• Tap any media item to play it on the LEDs
• Media plays in a loop by default
```

#### Audio tab
```
Configure the microphone for sound-reactive effects.

• Select your audio input device from the dropdown
• Tap "Start Audio" to begin listening
• Sensitivity adjusts how responsive effects are to sound
• Gain boosts quiet audio sources
• The meter bars show real-time bass, mid, and high frequency levels
• Sound-reactive effects won't respond to music until audio is started
```

#### Diagnostics tab
```
Test and troubleshoot your LED wiring.

• Wiring Tests: run patterns that help verify strip connections
  – Strip Identify: lights each strip one at a time (check numbering)
  – Bottom-Top Sweep: confirms vertical orientation
  – Serpentine Chase: verifies the zigzag wiring between paired strips
  – Seam Test: marks the wrap boundary between strip 9 and strip 0
  – Channel Identify: lights each electrical channel separately
  – RGB Order: sends red, green, blue to verify color wiring
• Teensy Tests: patterns generated by the Teensy hardware
• "Return to Normal" clears any test pattern and restores your scene
```

#### Sim tab
```
Preview animations without changing the live LEDs.

• Select an effect from the dropdown and tap "Preview"
• The pixel grid on the right shows what the animation looks like
• Each dot represents one LED — the layout matches your physical strip arrangement
• Adjust speed and palette in the controls on the left
• Preview runs independently from the live output
• Tap "Stop" to end the preview
• The preview disconnects automatically when you leave this tab
```

#### System tab — Status
```
System status and display settings.

• FPS Target: how many frames per second the renderer targets (60 is standard)
• Transport: shows if the Teensy hardware is connected
• Firmware: the version running on the Teensy
• Frames Sent: total frames delivered to the LEDs since startup
```

#### System tab — Setup
```
Configure your LED strip installation.

This is where you tell the controller about your physical LED hardware.
Changes here affect how animations are mapped to your strips.

• Start a Setup Session to begin editing (your current scene is saved and restored)
• Each strip row shows:
  – Label: a friendly name (e.g., "Front Left")
  – Enabled: whether this strip is active
  – LEDs: how many LEDs are installed on this strip (max 172)
  – Color: the native color byte order of the strip (usually BGR for WS2812B)
  – Direction: which way the strip is wired (↑ bottom-to-top or ↓ top-to-bottom)
  – Channel/Slot: which electrical output this strip is connected to
  – Chipset: the LED chip type
• Edit any field, then tap "Commit Changes" to save and apply
• Tap "Cancel" to discard all changes and restore your previous scene
• Changes take effect immediately — no restart required
```

#### System tab — Admin
```
System administration.

• Auth Token: set a bearer token to protect the API (leave empty for open access)
• Restart App: restarts the pillar controller service (takes ~5 seconds)
• Reboot Pi: reboots the entire Raspberry Pi (takes ~30 seconds)
```

## JS

```javascript
function initHelp() {
  document.querySelectorAll('.help-toggle').forEach(btn => {
    const panel = btn.closest('.help-panel');
    const content = panel.querySelector('.help-content');
    const tabId = panel.dataset.tab || 'unknown';

    // Restore state from localStorage
    const saved = localStorage.getItem(`help-${tabId}`);
    if (saved === 'open') {
      content.hidden = false;
      btn.setAttribute('aria-expanded', 'true');
      panel.classList.remove('collapsed');
    }

    btn.addEventListener('click', () => {
      const expanded = btn.getAttribute('aria-expanded') === 'true';
      btn.setAttribute('aria-expanded', !expanded);
      content.hidden = expanded;
      panel.classList.toggle('collapsed', expanded);
      localStorage.setItem(`help-${tabId}`, expanded ? 'closed' : 'open');
    });
  });
}
```

## Gate

- Every tab has a help panel
- Help is collapsed by default
- Collapse state persists across page loads
- Text is clear and actionable for first-time users
- No jargon without explanation
